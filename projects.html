Eklavya Sharma: Projects

  1. TheoremDep (Nov 2018 - Feb 2019)
    Topics: math, web development.

    • TheoremDep is a website that lists many theorems and shows the dependencies between them. Theorem X is said to be dependent on theorem Y iff Y is used as a lemma in the proof of X.
    • I created TheoremDep to better organize and document my own knowledge. TheoremDep mainly contains theorems on abstract algebra, linear algebra, number theory and graph theory.

  2. Internship at American Express, Gurgaon (Jan 2018 - June 2018)
    Topics: neural networks, machine learning, big data.

    • Trained a neural network from almost-raw data to estimate the probability of a credit-card applicant defaulting.
    • The data was in a unique format, so a custom neural network architecture was devised.
    • The neural network's performance was at par with the model then in production, which was tuned over many years and utilized several complex hand-engineered features.

  3. Analysis of Primality-testing Algorithms (Oct 2017 - Nov 2017)
    Advanced Algorithms and Complexity course project
    Topics: abstract algebra, number theory, computational complexity.

    • Studied the AKS primality-testing algorithm and attempted to improve it.
    • Compared compositeness-proving algorithms like Miller-Rabin, Solovay-Strassen, Baillie PSW, etc.

  4. Mitigating DNS-related DoS attacks using SDN (Sept 2017 - Dec 2017)
    Topics: computer networks, network security, SDN.

    • Studied DNS-related DoS attacks and Software-Defined Networking (SDN).
    • Devised a new mechanism for mitigating DNS amplification attacks, which uses a set of geographically-distributed SDN routers.
    • I presented a paper on it at ICACCI in September 2018.

  5. CT-means clustering algorithm (Nov 2017 - Jan 2018)
    Topics: machine learning, algorithms, math.

    • Invented a clustering algorithm, which I named CT-means. It is an approximation to C-means fuzzy clustering. It uses KD-trees to reduce running time.
    • Mathematically proved its convergence and approximation guarantees.
    • Implemented the algorithm and benchmarked its performance on different datasets. It was not significantly faster in practice and its applicability was limited.
